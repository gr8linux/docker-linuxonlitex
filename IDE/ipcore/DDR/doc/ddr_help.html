DDR

Information

Type: DDR
Vendor: GOWIN Semiconductor
Summary: Through this IP, users can produce a DDR, which is implemented by the combination of IOLOGICs, IOBUF, IODELAY and other primitives. DDR can be configured in different modes, including Input, Output, Tristate and Bidirectional modes. Users can also specify the Data Width, Ratio, and whether to use IODELAY and CLKDIV.
Devices: GW1N-1, GW1NR-1, GW1NZ-1, GW1NZ-1C, GW1N-1S, GW1N-2, GW1NR-2, GW1N-4, GW1N-4B, GW1N-4D, GW1NR-4, GW1NR-4B, GW1NR-4D, GW1NRF-4B, GW1NS-2, GW1NS-2C, GW1NSR-2, GW1NSR-2C, GW1NSE-2C, GW1NS-4, GW1NS-4C, GW1NSR-4, GW1NSER-4C, GW1NSR-4C, GW1N-9, GW1N-9C, GW1NR-9, GW1NR-9C, GW2A-18, GW2AR-18, GW2A-18C, GW2AR-18C, GW2ANR-18C, GW2A-55, GW2A-55C, GW2AN-55C, GW2AN-18X, GW1N-1P5

Options

DDR Mode

  • Allows you to select the DDR mode, including Input, Output, Tristate and Bidirectional.

Data Width

  • Allows you to specify the DDRâ€™s data width.

Ratio

  • Specify the data conversion ratio, including 2, 4, 7, 8, 10 and 16. 2 means data 1:2 or 2:1 conversion, 4 means data 1:4 or 4:1 conversion, others are similar.

Reset

  • Selecting this option will produce IDDRC or ODDRC instance in the generated module.

IODELAY

  • Delay Mode - If you do not want to use IODELAY, you can choose NONE, otherwise choose Dynamic or Static.
  • Delay Step - Allows you to enter the value of delay steps when Delay Mode is Static mode.
  • Delay Direction - Choose whether IODELAY is connected to input or output when DDR Mode is Bidirectional mode.

Use CLKDIV

  • Selecting this option will produce CLKDIV instance in the generated module.
